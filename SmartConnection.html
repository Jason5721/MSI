Smart Connection

Intro

Welkom bij mijn persoonlijke beleving van de module Smart Connection. In deze module volg ik een aantal lessen die bijdragen aan de ontwikkeling van mijn persoonlijke website. Ik leer bijvoorbeel hoe ik een website moet vormgeven en wat voor achterliggende processen deze heeft. Gedurende dit traject wordt er uitgelegd hoe ik met html, css en js moet werken en wat deze met je website doen.

Ik heb eerder nog nooit zelfstandig een website gebouwd, dus ik had deelgenomen aan alle lessen. Hoop op deze manier voldoende kennis op te doen en hiermee een prachtige website op het einde van de minor kan opleveren.

Week 1-Verkenning

Intro

In deze week waren wij gestart met kennismaking met html(Hyper Text Markup Language), een programmeertaal die wordt gebruikt om websites op te bouwen. Hierover zijn een aantal basisaspecten behandeld en zijn een aantal opdrachten mee uitgevoerd.

Uitvoering

De les stond in het teken van wat theoriestof (de basis van html) en het maken van een grote opdracht. Deze opdracht bestond uit het aanpassen van een pagina naar eigen versie. ik heb ervoor gekozen om een pagina van IKEA aan te passen en hiervan mijn eigen profiel te maken. Het lukte me al vrij snel om de naam van het product aan te passen naar mijn naam en hiervoor ook een leuke "prijs" erbij zetten. Maar afbeeslding vervangen door mijn eigen foto was helaas niet gelukt.

Reflectie

Het was voor mij een kleine uitdaging omdat ik eerder nog nooit met html heb gewerkt. Maar het lukte me al vrij snel om nieuwe stof op te pakken.

Week 2-Web Dev HTML

Intro

Deze week van de module staat in het teken van een stukje verdieping in het bouwen van een website, namelijk kennismaken met css. Zelfde als met html, heb ik eerder nog nooit gewerkt met deze programmeer/configuratie talen. Ook wordt er kennisgemaakt met GitHub, wat een manier is om je eigen website online te hosten.

Uitvoering

Ik ben eerst begonnen met het opzoeken van informatie over GitHub. Ik was dit filmpje tegengekomen waarin GitHub wordt uitgelegd aan de hand van een praktijkvoorbeeld.

Nadat de betekenis GitHub voor mij verduidelijkt is, heb ik mezelf verdiept in css

Reflectie

Vanuit mijn opleiding heb ik nooit gewerkt met github (omdat wij ons meer verdiepen in procesautomatisering). Daarom was het voor mij zeer leerzaam om met GitHub te werken. Dit omdat de software ingewikkelde versiebeheersystemen voorkomt en gemakkelijk te begrijpen is (als de persoon enige ervaring heeft met softwareontwikkeling). Daarnaast vond ik website bouwen met html en css heel leerzaam.

Week 3-Web Dev Javascript

Intro

Javascript is voor mij een taal die ook nog niet bekend was. Dus had me meer moeite gekost om het te begrijpen en tegelijkertijd bijhouden met de stof die wordt uitgelegd.

Uitvoering

Tijdens de les hadden we geëxperimenteerd met Javascript. Wij hadden een hulpprogramma gebruikt genaamd CodePen wat heel erg behulpzaam was. Daarmee had ik wat codes gevonden voor bijvoorbeeld een klok of een weer weergave. Nadat ik de codes van de CodePen had geïmplementeerd, verscheen een klok op mijn webpagina. Daarna heb ik nog een aantaal andere handige features van CodePen kunnen toepassen op mijn eigen website.

Reflectie

De stof die behandeld is in deze week sluit niet helemaal aan op mijn verwachtingen, Maar ik vond het toch heel leerzaam om deze programmeertaal toe te passen om een website vorm te geven. Dit wil ik later graag toepassen om mijn eigen website te creëren.

Week 4-Databases

Intro

Verrassend genoeg heb ik eerder wel wat kennisgemaakt met database en data verwerken i.v.m mijn afstudeerstage. Wij moesten toen alle gemeten datas opslaan in een database en aan de hand van de vraag wordt er informatie teruggehaald uit de database. Maar ik verwacht dat er meer in zit dan wat ik toen had gedaan met mijn afstudeerpartner.

Uitvoering

Tijdens de les heeft Witek een leuke experiment uitgevoerd samen met de studenten. Het ging asl volgt: Hij hield drie pennen voor zijn laptop met een camera vast met drie verschillende kleuren, namelijk rood, blauw en groen. Bij elke pen had hij meer dan 1000 foto's opgenomen met de camera en deze wordt opgeslagen in een database. Vervolgens worden de informaties die worden opgeslagen in database verwerkt. En de volgende keer hij een pen voor de camera vasthield, wist het systeem hem te vertellen welke kleur deze pen heeft.

Reflectie

Ik vond deze les zeer interessant. Omdat de les interactief was, heb ik actief meegedaan en aandachtig geluisterd naar de uitleg die hij gaf. En hierdoor kwam ik achter dat er andere manieren zijn om met data te werken, anders dan wat ik met mijn afstudeerstage had gedaan.

Week 5-APIs

Intro

APIs, voor mij een totaal onbekende wereld. Voorheen heb ik ooit wel eens van de naam gehoord, maar verder had ik geen idee wat deze doet. Mijn doel deze week is daarom ook om APIs te ontdekken en toe te kunnen passen op mijn portfolio.

Uitvoering

Voorafgaand aan de les heb ik het korte introductiefilmpje bekeken over API's. In mijn opzicht is een API een interface voor andere websites om gebruik te maken van functies van andere websites. Hierdoor kan gemakkelijk data gebruikt worden van externe partijen.

Gedurende de les was een verhelderende uitleg over API's. Hieruit kan ik concluderen dat mijn verwachting over de functie van API's wel klopt. Gedurende de les hebben we wat voorbeelden gehad en zijn we vervolgens gestart met een aantal opdrachten. De opdrachten bestonden uit een aantal verbindingen met het coronadashboard en het weer.

Reflectie

Ik vond deze lesstof zeer leerzaam. Met name de mogelijkheden die API's bieden spreken mij zeer aan. Dit vanwege de gereduceerde ontwikkeltijden die het met zich meebrengt wanneer er gebruik wordt gemaakt van dit mechanisme. Daarnaast vind ik het goed concept dat meerdere partijen samenwerken voor een optimale website. Voor het business aspect kan dit dus veel schelen aan ontwikkelkosten.

Week 6-Big Data & Machine Learning

Intro

Deze les is wat meer theoretisch dan praktisch. Hierin wordt globaal uitgelegd hoe Big Data & Machine Learning wordt toegepast in de praktijk. Deze uitleg wordt verricht aan de hand van een aantal praktijkvoorbeelden. Voor het Big Data onderdeel was dit een algemene uitleg. Bij Machine Learning uitleg over het samenstellen van een algoritme. Er was voor deze samenstelling gekozen omdat Machine Learning Big Data gebruikt om een uitkomst te genereren.

Uitvoering

Zoals al eerder benoemd is de les begonnen met een stukje theoriestof. In deze theoriestof zijn twee verschillende uitkomstmogelijkheiden besproken: deterministisch en storgastisch. Deterministische gegevens staan hierbij vast en Storgastische gegevens zijn vaak gebaseerd op een willekeurige uitkomst.

In de afbeelding is al zichtbaar dat bij het genereren van de curce gebruik is gemaakt van een grote hoeveelheid data. Hier komt dan ook het onderdeel Big Data bij kijken. Om een Machine Learning model goed te laten functioneren is big data nodig.

Reflectie

In de praktijk kan het gebruik van deze algoritmes veel mogelijkheden opleveren. Denk hierbij aan algoritmes die kunnen bepalen waar een object ligt in een 3d veld door positie te matchen aan de hand van een pointcloud. Naast het technische toepassingsgebied van Machine Learning zijn er overal mogelijkheden te vinden. Hiervan is de medische benadering zeer interessant. Bij deze benadering wordt gebruik gemaakt van een Machine-Learning model om een diagnose te stellen. Dit biedt veel ondersteuning aan doktoren.

Op persoonlijk gebied heb ik mijn algemene kennis over Big Data en Machine Learning vergroot. Daarnaast ben ik zeer geïntresseerd in het onderwerp en hoop ik dat ik mij kan verdiepen in mijn toekomstigge carriere.
